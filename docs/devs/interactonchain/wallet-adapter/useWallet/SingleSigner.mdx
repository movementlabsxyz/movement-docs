# All Signer Function Working with MoveVM Wallet
Commonly used in blockchain applications to prove ownership of a wallet address by single signing messages

- Getting it signed by the wallet
- Making sure the signature is legit

```typescript
const onSignTransaction = async () => {
  try {
    const payload = {
      type: "entry_function_payload",

    console.error(error);
  }

```

## 4ï¸âƒ£ Sign Transaction V2

```typescript
const onSignTransactionV2 = async () => {
  if (!account) return;
  try {
    const transactionToSign = await aptosClient(network).transaction.build.simple({
      sender: account.address,
      data: {
        function: "0x1::coin::transfer",
        typeArguments: [APTOS_COIN],
        functionArguments: [account.address, 1], // 1 Octa
      },
    });
    const response = await signTransaction(transactionToSign);
    toast({
      title: "Success",
      description: JSON.stringify(response),
    });
  } catch (error) {
    console.error(error);
  }
};
```

This one's pretty straightforward:
- Creates a simple token transfer transaction
- Gets it signed by your wallet
- Shows a success message when everything's good


## 5ï¸âƒ£ Sign and Submit BCS Transaction

```typescript
const onSignAndSubmitBCSTransaction = async () => {
  if (!account) return;

  try {
    const response = await signAndSubmitTransaction({
      data: {
        function: "0x1::coin::transfer",
        typeArguments: [parseTypeTag(APTOS_COIN)],
        functionArguments: [AccountAddress.from(account.address), new U64(1)],
      },
    });
    await aptosClient(network).waitForTransaction({
      transactionHash: response.hash,
    });
    toast({
      title: "Success",
      description: <TransactionHash hash={response.hash} network={network} />,
    });
  } catch (error) {
    console.error(error);
  }
};
```

This is the cool one! It:
- Signs AND submits your transaction in one go
- Uses BCS (fancy way of encoding data that's more efficient)
- Waits for confirmation
- Shows you a clickable transaction hash

ðŸ’¡ **What makes this one special?**
- It's a one-stop-shop: signs and submits in one function
- Uses BCS format (smaller size = lower fees)
- Makes sure your transaction is confirmed before saying "success"
- Gives you a link to check your transaction on the explorer
